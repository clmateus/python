# Programação Orientada a Objetos

A POO é um paradigma de programação.

Uma de suas grandes vantagens é que ela permite e incentiva o reuso de código, ou seja, tenta buscar resolver vários problemas utilizando um mesmo conjunto de códigos.

## Classe

Classes são representações de um elemento, real ou não. Veja abaixo o exemplo de uma classe que representa uma pessoa genérica:

Classe "Pessoa"
Atributos
- Nome
- Idade
- Documento
Métodos
- Andar
- Comer
- Dormir

Toda classe pode possuir atributos e métodos. Atributos são características individuais, enquanto métodos são ações que a classe pode realizar.

## Objeto

Objeto é a instância de uma classe. Veja abaixo o exemplo de um objeto que se origina da classe "Pessoa":

Objeto instância da classe "Pessoa"
Atributos
- Nome: "João Silva"
- Idade: 25
- Documento: 123.456.789-01

Usando objetos podemos criar várias instâncias de classes genéricas que podem ser reaproveitadas inúmeras vezes.

## Herança

Herança é a especialização de uma classe. Pensando no exemplo acima, "Estudante" poderia ser uma especialização da classe "Pessoa", já que um estudante teria todos os atributos e métodos comuns de uma classe, mas com adicionais que nem toda pessoa teria.
